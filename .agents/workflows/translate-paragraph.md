# /translate-paragraph

마크다운 파일을 받아 한국어 번역이 없는 영어 단락을 자동으로 찾아 번역하는 워크플로우입니다.
이 워크플로우는 **Antigravity** 에이전트에 의해 실행됩니다.

## 주요 기능
1. 한국어 번역 없는 영어 단락 자동 감지 (영어 단락 바로 아래에 한국어가 없는 경우 감지)
2. 각 단락의 한국어 번역 생성
3. 번역 품질 검사 (의미 정확성, 자연성, 마크다운 보존, 전문 용어 일관성)
4. 문제 발견 시 자동 재번역 및 검증
5. 모든 번역이 완료된 후 원문 바로 아래에 번역 문장 삽입

## 사용법

다음과 같이 슬래시 커맨드와 함께 파일이나 폴더 경로를 지정합니다:

**단일 파일 번역**:
```
/translate-paragraph "markdown/7. Inductive-Types.md"
```

**폴더 전체 번역** (주의: 폴더 내의 파일들을 순차적으로 처리합니다):
```
/translate-paragraph "markdown"
```

## 번역 규칙

- **전문 용어**: 영어 원문 유지 (예: API, type theory, Antigravity)
- **마크다운 보존**: 코드블록, 링크(`[]()`), 강조(`**`, `_`) 등 모든 관련 문법 유지
- **스타일**: 자연스러운 한국어, 기술 및 학술적 톤 유지

## 검사 항목 (Antigravity 자체 점검 목록)

Antigravity는 번역 후 다음 항목을 스스로 검증합니다.
✅ **의미 정확성** - 원문과 번역의 의미 동일 여부
✅ **자연스러운 표현** - 한국어의 자연스러움
✅ **마크다운 보존** - 코드블록, 링크 등 문법 유지
✅ **전문 용어 일관성** - 프로젝트 전반 용어 통일

검사 중 문제 발견 시 내용을 스스로 교정(최대 2회)한 뒤 파일에 작성합니다.

## Antigravity의 작업 지침

사용자가 `/translate-paragraph <경로>` 명령을 사용하면, Antigravity는 다음 지침을 따릅니다:

1. **경로 확인 및 파일 읽기**:
   - 단일 파일인 경우 해당 마크다운 파일을 읽습니다.
   - 폴더인 경우 폴더 내의 `.md` 파일들을 찾아 목록화하고, 한 파일씩 순차적으로 진행합니다.
2. **단락 감지 및 구조 분석**:
   - 파일에서 이미 한국어 번역이 있는 단락과 번역이 누락된 영어 단락들을 식별합니다.
   - 영어 단락 다음에 한글 단락이 없다면 번역을 수행해야 할 대상으로 판단합니다.
3. **번역 수행 및 퀄리티 체크**:
   - 각 대상 단락을 번역 규칙에 따라 한국어로 번역합니다.
   - 4가지 검사 항목을 스스로 체크하며, 부족하다면 재번역합니다.
4. **파일 수정 (`multi_replace_file_content` / `replace_file_content` 활용)**:
   - 번역이 완료되면 원본 파일의 정확한 위치(대상 영어 단락 바로 아래)에 번역된 내용을 추가합니다.
5. **결과 보고**:
   - 파일(또는 폴더 내 전체 파일) 작업 완료 시 어떤 부분이 번역되었는지 사용자에게 간략히 요약 보고합니다.
